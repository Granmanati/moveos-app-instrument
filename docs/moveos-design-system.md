Version 1.0  
Status: Production Reference  
Owner: MOVE OS Product Team

# 1. Brand Foundation

MOVE OS is a technology company designing adaptive systems for human movement.

It is not:

• a fitness app  
• a wellness platform  
• a medical interface  
• a workout library  

MOVE OS is:

**a movement operating system**

The visual system must communicate:

• technological precision  
• structural intelligence  
• calm authority  
• future-oriented product design  

The interface must feel closer to:

Stripe  
Linear  
Apple  
Notion  

than to fitness or lifestyle applications.

---

# 2. Core Design Principles

All UI decisions must respect the following principles.

### 2.1 System before decoration

Every visual element must communicate system structure or function.

Avoid decorative elements without functional meaning.

---

### 2.2 Quiet power

MOVE OS is visually restrained.

No excessive effects.
No visual noise.

Hierarchy and whitespace carry meaning.

---

### 2.3 Precision

Typography, spacing and alignment must feel engineered.

UI must communicate:

control  
structure  
intelligence

---

### 2.4 Identity over intensity

MOVE OS builds behavioral identity through system continuity.

The design must reinforce:

consistency  
return  
adaptation

---

# 3. Brand Symbol System

The MOVE OS symbol represents:

**System + Node + Line**

Meaning:

• system architecture  
• adaptive pathways  
• movement intelligence  

The symbol should appear in:

• application icon  
• system header  
• institutional surfaces

Avoid:

• large decorative use
• repeated watermarking
• background tiling

The symbol must feel like a **software mark**, not decoration.

## 4.2 Functional State Colors

MOVE OS introduces a **Health State System**.

These colors represent system conditions.

Success / Aligned  
#18B67A

Warning / Compensating  
#E8A23A

Alert / Overload  
#E45462

Rules:

• state colors must appear only when meaningful  
• never use them for decorative purposes  
• maintain subtlety and control

# 6. Grid and Spacing

MOVE OS uses a strict spacing system.

Base grid: **8px**

Spacing tokens:

8  
16  
24  
32  
48  
64

Rules:

• maintain vertical rhythm  
• avoid arbitrary spacing  
• large whitespace is intentional

---

# 7. Structural Interface Layer

MOVE OS UI includes a subtle **structural overlay**.

Purpose:

communicate system intelligence.

Elements may include:

• subtle grid lines  
• node connection points  
• thin technical dividers  
• micro system labels

Example elements:

• connection lines
• corner frame segments
• technical annotations

Rules:

• must remain subtle
• never dominate UI
• reinforce system feeling

---

# 8. UI Surface Style

Cards and panels follow these rules.

Background

#141A22

Border

1px solid #1C2430

Corner radius

12–16px

Shadow

Minimal or none

Cards must feel:

• engineered
• calm
• precise

---

# 9. Motion Philosophy

Animation should communicate system behavior.

Allowed motion:

• lines drawing in
• node activation
• progress indicators
• subtle fades

Avoid:

• bounce animations
• playful motion
• heavy transitions

Motion should feel:

technical  
controlled  
intentional

---

# 10. Product Identity

MOVE OS should visually communicate:

**a system managing human adaptation**

The UI must feel:

• intelligent  
• reliable  
• premium  

Never:

• motivational
• gamified
• fitness influencer aesthetic
