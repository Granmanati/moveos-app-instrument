# MOVE OS UI Rules
Version 1.0

This document defines non-negotiable UI constraints.

These rules exist to preserve visual integrity.

---

# 1. Non-Negotiable Rules

The following patterns are prohibited.

No bright gradients  
No neon glow effects  
No fitness influencer visuals  
No wellness pastel palettes  
No emoji-style icons  
No oversized shadows  

The UI must never resemble:

• fitness apps
• lifestyle coaching apps
• wellness dashboards

---

# 2. Visual Tone

MOVE OS UI must feel:

technical  
minimal  
calm  
precise

Comparable references:

Stripe Dashboard  
Linear Interface  
Apple System Apps

---

# 3. Color Usage Rules

Blue (#2D7CFF) is the system accent.

Rules:

• never exceed 10–15% of interface area  
• use mainly for active states and highlights  

State colors:

Green → success state  
Amber → compensation state  
Red → overload state

These must appear only when meaningful.

---

# 4. Typography Rules

Inter for primary text.

IBM Plex Mono for:

• metrics
• system indicators
• technical labels

Example:

Return Rate  
86%

---

# 5. Layout Hierarchy

Screen hierarchy must follow:

1 System State  
2 Active Mission  
3 Key Metric  
4 Supporting Data

This ensures the user always understands:

system status first.

---

# 6. Card System

Cards represent system modules.

Each card must contain:

• clear label
• meaningful metric
• clear action

Cards must never be decorative.

---

# 7. Logo Usage

MOVE OS symbol should appear:

• header
• application icon
• key system moments

Never:

• repeated watermark
• background decoration

---

# 8. Visual Density

MOVE OS favors:

clarity over density.

Avoid:

crowded dashboards  
too many widgets  
micro metrics everywhere

---

# 9. System Language

Labels must feel technical.

Examples:

Return Rate  
Session Load  
System Status  
Adaptive Mode

Avoid:

motivational language.

---

# 10. Product Credibility

The interface must communicate:

real system  
real intelligence  
real product

Not a concept.
